El código como sistema visual

El enfoque de Casey Reas se basa en la creación de instrucciones que definen comportamientos visuales. En lugar de diseñar una imagen de forma directa, desarrolla algoritmos que determinan cómo los elementos se relacionan y evolucionan en el tiempo.

Sistemas, reglas y variación

El trabajo de Casey Reas se fundamenta en la idea de que una obra puede ser entendida como un sistema de reglas. Este enfoque introduce la variación como un elemento central: cada vez que el sistema se ejecuta, puede producir un resultado diferente.

El rol del artista-programador

En la práctica de Reas, el artista asume el rol de programador. La autoría se desplaza hacia el diseño del sistema, mientras que la ejecución introduce elements de autonomía y sorpresa en la obra.
